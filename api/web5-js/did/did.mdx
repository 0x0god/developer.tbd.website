## Methods

### create(method, options)

Enables generation of DIDs for a supported set of DID Methods. The output is method-specific, and handles things like key generation and assembly of DID Documents that can be published to decentralized DID networks.

#### Parameters

`method`: The method on which to create the did. Supported methods are _ion_, _web_, and _key_

`options` _(Optional)_: Enables customization of the did formation

#### Return Value

Did document

#### Code Example

```js
const myDid = await web5.did.create('ion');
```

### encrypt(options)

Encrypts provided data according to specification

#### Parameters
`options`: Object containing:

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; `did`: DID of the recepient, whose private key is used to decrypt the data

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;`payload`: Encryption algorithm output, including the ciphertext, to be decrypted

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;`keyId`: Key identifier to use form the recepient's DID document

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;`algorithm`: An object specifying the algorithm to be used and any extra parameters if required

#### Return Value
`Promise<Web5EncryptionResult>`: A promise that fulfills with the output of the encryption algorithm

#### Code Example
```js
const options = {
    "did" : "firstDid",
    "payload" : "decodePayload"
    "keyId" : "sampleKeyId",
    "algorithm' : "sampleAlgorithm"
}
const encryptedText = await web5.did.encrypt(options)
```
